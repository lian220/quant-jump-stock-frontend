# 전환 차단 이슈(P0) 정리 및 확정 범위

## 목적
- 공개 라우트에서 가입 전환을 막는 즉시 결함을 제거한다.
- 사용자에게 로그인/회원가입/마이페이지 진입 경로를 일관되게 제공한다.

## 확정 범위
- `회원가입 링크`는 모든 주요 진입점에서 실제 가입 경로(`/signup`)로 연결
- `MY` 진입은 비로그인 시 `/auth`, 로그인 시 `/mypage`로 라우팅
- 구 `/my` 접근은 `/mypage`로 리다이렉트 처리

## 반영 상태
- `Header` 데스크톱/모바일에 `회원가입` CTA 추가 및 `/signup` 연결
- 랜딩 주요 CTA 문구를 `무료 회원가입`으로 통일하고 `/signup` 연결
- `src/app/my/page.tsx`에서 `/mypage` 리다이렉트 유지

## QA 체크리스트
- 비로그인 상태에서 헤더 `회원가입` 클릭 시 `/signup` 진입
- 비로그인 상태에서 하단 `MY` 탭 클릭 시 `/auth` 진입
- 로그인 상태에서 하단 `MY` 탭 클릭 시 `/mypage` 진입
- 주소창에 `/my` 직접 입력 시 `/mypage`로 이동
