# Alpha Foundry 메인 화면 전문가 종합 리뷰

> 작성일: 2026-03-10
> 대상: `https://alphafoundry.app`
> 검토 범위: 비로그인/로그인, 데스크톱/모바일 메인 화면
> 검토 방식: 실제 브라우저 E2E 확인 + UX 프레임워크 기반 전문가 리뷰

## 핵심 진단
- 가장 큰 문제는 "정보 부족"이 아니라 "우선순위 부재"다.
- 비로그인 화면은 CTA가 분산되어 있고, 로그인 화면은 데이터가 많지만 다음 행동이 선명하지 않다.
- 모바일은 현재 치명적인 반응형 문제로 인해 퍼스트뷰 신뢰와 사용성이 무너진 상태다.
- AI 투자 플랫폼으로서 강점이 될 수 있는 개인화, 설명 가능한 데이터, 첫 아하 모먼트 설계가 메인 화면에서 충분히 드러나지 않는다.

## 실제 관찰 요약
### 비로그인 데스크톱
- 히어로 카피는 `주식, 뭘 사야 할지 모르겠다면`으로 직관적이다.
- 하지만 `로그인`, `회원가입`, `오늘의 AI 추천 보기`가 동시에 강하게 보여 1차 행동이 분산된다.
- 히어로 아래에 AI 주목 종목, 뉴스, 전략, 하단 회원가입 CTA가 이어지지만 "왜 지금 가입해야 하는지"에 대한 설득 구조는 약하다.

### 비로그인 모바일
- 헤더 링크 4개와 인증 버튼이 한 줄에 그대로 노출되어 첫 화면 대부분을 차지한다.
- 히어로 문구가 잘리고 가로 오버플로우처럼 보여 서비스 완성도에 대한 신뢰를 크게 떨어뜨린다.
- 핵심 CTA보다 내비게이션이 먼저 시야를 점유한다.

### 로그인 데스크톱
- 로그인 후 `오늘의 AI 분석`과 요약 수치가 노출되어 비로그인 대비 목적성은 조금 더 분명해진다.
- 그러나 `전체 분석 보기`, `전략 둘러보기`, 알림, 계정 정보, 수치 요약이 동시에 경쟁해 시선이 분산된다.
- `74%`, `72점` 같은 숫자는 보이지만 정의와 기준 시점이 부족하다.

### 로그인 모바일
- 로그인 후에도 데스크톱형 헤더 구조가 유지되어 상단 공간을 과도하게 차지한다.
- 제목과 요약 카드가 퍼스트뷰에서 잘리거나 어색하게 배치되어 핵심 인사이트가 즉시 전달되지 않는다.
- 신규 가입자, 구독 0, 알림 0인 사용자에게 무엇을 먼저 해야 하는지 안내가 없다.

## 1. Jakob Nielsen 관점
### 핵심 분석
- `시스템 상태 가시성`이 약하다. 핵심 수치의 기준 시점, 갱신 주기, 의미가 없다.
- `현실 세계와의 일치`가 부족하다. AI 점수와 비율이 투자자 언어로 번역되지 않는다.
- `심미성과 미니멀리즘` 원칙이 깨져 퍼스트뷰에서 헤더, CTA, 카드가 동시에 경쟁한다.
- 모바일 레이아웃 붕괴는 사용성 문제가 아니라 신뢰 훼손 문제이기도 하다.

### 구체적 액션 아이템
- 모든 핵심 수치에 `무엇을 의미하는지`, `언제 기준인지`, `왜 중요한지`를 붙인다.
- 비로그인 퍼스트뷰 CTA를 1개로 압축하고 나머지는 보조 경로로 낮춘다.
- 로그인 메인 첫 화면은 `오늘 가장 먼저 봐야 할 것` 1개만 강조한다.
- 모바일 헤더를 `로고 + 메뉴 + 1차 CTA` 수준으로 축소한다.

### 와이어프레임 반영
- 로그인 메인의 상단 수치 카드를 2개 이하로 줄이고, 수치 아래 해석 문장을 추가한다.

## 2. Steve Krug 관점
### 핵심 분석
- 첫 방문자가 "지금 뭘 해야 하지?"를 생각하게 만든다.
- 비로그인 화면은 `로그인`, `회원가입`, `오늘의 AI 추천 보기`가 서로 경쟁한다.
- 로그인 화면은 `전체 분석 보기`와 `전략 둘러보기`가 상단에 동시에 떠 있어 주 행동이 흐려진다.
- 5초 테스트 기준으로 "AI가 오늘 뭘 추천했고 왜 중요한지"는 충분히 전달되지 않는다.

### 구체적 액션 아이템
- 비로그인 화면의 핵심 질문을 `오늘 AI가 본 종목을 바로 확인할까?`로 통일한다.
- 로그인 화면의 핵심 질문을 `내가 지금 확인해야 할 1순위는 무엇인가?`로 재정의한다.
- 히어로는 `한 줄 가치 제안 + 한 줄 설명 + 주 CTA 1개` 구조로 고정한다.
- 섹션 미리보기는 3개 이하만 보여주고 나머지는 더보기로 위임한다.

### 와이어프레임 반영
- 상단 CTA 버튼은 주 CTA 1개, 보조 CTA 1개만 남기고 나머지는 아래 섹션으로 이동한다.

## 3. Luke Wroblewski 관점
### 핵심 분석
- 현재 모바일은 "퍼스트"가 아니라 데스크톱 화면의 축소판에 가깝다.
- 정보 밀도가 아니라 구조적 붕괴가 문제다.
- AI 추천 서비스라면 토스증권처럼 데이터만 밀어 넣는 방식보다, Robinhood식 극단 단순화도 아닌 "요약 중심 중간 지점"이 적합하다.
- 모바일 첫 화면은 시장 전체보다 `오늘의 핵심 추천 1개` 혹은 `나를 위한 요약`을 먼저 보여줘야 한다.

### 방향 제안
- 방향성은 `Robinhood의 단순 진입성 + 토스증권의 신뢰형 데이터 설명`의 절충안이 적합하다.
- 첫 화면에는 1개의 핵심 요약 카드와 1개의 주 CTA만 둔다.
- 뉴스, 전략, 세부 종목 카드는 첫 화면 아래로 내린다.
- 내비게이션은 숨기고, 사용 빈도가 높은 탐색만 하단 탭 또는 메뉴로 분리한다.

### 모바일 첫 화면에서 보여줄 것
- 오늘의 AI 한 줄 요약
- 대표 추천 종목 1~3개
- 왜 추천했는지 한 줄 이유
- 주 CTA 1개

### 모바일 첫 화면에서 숨길 것
- 전체 메뉴 링크 나열
- 전략 카드 대량 노출
- 뉴스 목록 우선 노출
- 로그아웃 같은 저빈도 액션의 상단 고정 노출

### 와이어프레임 반영
- 모바일 전용 헤더와 전용 퍼스트뷰 레이아웃을 별도로 설계한다.

## 4. Edward Tufte 관점
### 핵심 분석
- 숫자는 있지만 맥락이 부족해서 정보가 인사이트가 되지 못한다.
- `74%`, `72점`은 강조되지만 비교 기준과 추세가 없어 해석 비용이 높다.
- 종목 카드마다 반복되는 배지와 지표가 많아 시각적 노이즈가 크다.
- 투자 서비스라면 화려한 카드보다 "왜 이 종목이 중요한가"를 더 선명하게 보여줘야 한다.

### 구체적 액션 아이템
- `전일 대비`, `주간 대비`, `시장 평균 대비` 같은 비교 기준을 함께 표기한다.
- 종목 카드 상단에는 핵심 근거 1줄만 남기고 나머지는 상세 화면으로 위임한다.
- 로그인 메인에 미니 스파크라인을 도입하되, 축약 차트는 장식이 아니라 추세 전달에만 쓴다.
- 불필요한 차트 장식, 배지 남발, 중복 라벨은 제거한다.

### 트랙레코드 미니 차트 제안
- 세로 높이 24~32px의 단일 선형 sparkline
- 축, 격자, 범례 제거
- 기준점은 최근 30일 혹은 최근 12회 추천 성과 중 하나로 통일
- 라인 색은 상승/하락만 구분하고, 숫자 라벨은 우측 끝 1회만 표기

### chartjunk 제거 대상
- 동일 카드 내 반복되는 `차트 패턴`, `AI 예측`, `뉴스 반응` 박스 과다 노출
- 의미 설명 없는 배지 강조
- 같은 정보의 텍스트와 숫자 중복 표기

### 와이어프레임 반영
- 로그인 요약 영역을 `오늘의 AI 브리프` 카드 1장과 보조 지표 1~2개로 압축한다.

## 5. Nir Eyal 관점
### 핵심 분석
- 비로그인 트리거는 있으나 개인 관련성과 긴급성이 약하다.
- 행동 유도는 많지만 한 번에 하나의 행동으로 수렴되지 않는다.
- 추천이 반복 노출되지만 개인화된 보상이 약해 습관 형성에 불리하다.
- 신규 사용자의 투자 성향 입력, 관심 종목 등록, 알림 설정 같은 `Investment` 단계가 없다.

### 구체적 액션 아이템
- 비로그인 첫 화면에 `오늘 AI가 가장 강하게 본 종목 3개`를 노출한다.
- 로그인 사용자는 `내 관심 섹터`, `최근 본 종목`, `내 알림` 기반 홈으로 전환한다.
- 신규 가입 직후 `관심 섹터 선택`, `위험 성향 선택`, `알림 받기` 3단계 온보딩을 제공한다.
- 첫 번째 아하 모먼트는 `내가 이해할 수 있는 추천 1개를 받고 이유를 바로 보는 경험`으로 정의한다.

### 신규 가입자의 빈 대시보드 해법
- 구독 0, 알림 0, 사용량 0 상태에서는 빈 화면 대신 시작 가이드를 보여준다.
- 기본값으로 전체 시장 추천을 채워 넣되, 사용자의 입력을 통해 빠르게 개인화로 전환한다.
- 강제 온보딩보다 `기본값 채움 + 선택형 세부 설정` 전략이 더 적합하다.

### 와이어프레임 반영
- 로그인 직후 빈 상태용 `시작하기` 카드와 관심사 설정 플로우를 메인 상단에 추가한다.

## 6. Julie Zhuo 관점
### 핵심 분석
- 개편 방향은 많지만 어떤 가설을 어떤 지표로 검증할지가 아직 선명하지 않다.
- 비로그인, 신규 가입자, 재방문 로그인 사용자의 목적이 다른데 홈이 충분히 분리되어 있지 않다.
- 메인은 추천 플랫폼이 아니라 정보 포털처럼 느껴질 위험이 있다.

### A/B 테스트 핵심 가설 3가지
1. 비로그인 메인에서 `오늘의 AI 추천 보기` 단일 CTA만 강조하면 기존 구조보다 1차 클릭률과 회원가입 전환율이 올라갈 것이다.
2. 로그인 메인을 `카드 나열형`에서 `오늘의 AI 브리프 + 개인화 리스트` 구조로 바꾸면 `전체 분석 보기` 진입률과 핵심 기능 클릭 수가 올라갈 것이다.
3. 신규 가입자에게 첫 방문 시 `관심 종목/섹터 설정` 온보딩을 노출하면 7일 재방문율과 알림 설정률이 올라갈 것이다.

### 정성 검증이 필요한 항목
- 히어로 카피 이해도
- 핵심 수치의 해석 가능성
- 모바일 첫 화면에서 느끼는 신뢰도
- 신규 가입자의 첫 아하 모먼트 도달 속도

### 정량 검증이 필요한 항목
- 메인 1차 CTA 클릭률
- 로그인 후 핵심 분석 진입률
- 신규 가입 후 관심사 설정 완료율
- 7일 재방문율
- 알림 설정 전환율

### 와이어프레임 반영
- 홈 모듈별로 KPI를 정의하고, 레이아웃 변경안을 지표 실험 단위로 쪼개서 검증한다.

## 우선순위 높은 수정사항
1. 모바일 헤더 반응형 붕괴를 즉시 수정한다.
2. 비로그인 퍼스트뷰 CTA를 1개로 통합한다.
3. 로그인 퍼스트뷰의 수치와 CTA를 줄여 `오늘의 핵심 액션`만 남긴다.
4. 핵심 숫자에 의미, 기준 시점, 비교 기준을 추가한다.
5. 신규 가입자/빈 상태 전용 시작 가이드를 만든다.
6. 뉴스와 전략은 메인 상단 우선순위에서 한 단계 낮춘다.
7. 종목 카드의 반복 지표를 정리하고 트랙레코드 미니 차트로 압축한다.

## 와이어프레임에 바로 반영할 수정안
1. 비로그인 히어로: `문제 제기 1줄 + 해결 약속 1줄 + CTA 1개`
2. 비로그인 메인: `오늘의 AI 추천 3종목`을 히어로 바로 아래 배치
3. 로그인 메인: `오늘의 AI 브리프` 단일 카드 우선 배치
4. 로그인 메인: `내가 지금 볼 것` 섹션을 종목, 뉴스, 전략보다 상단에 둠
5. 모바일: 햄버거 메뉴 도입, 인증/보조 액션은 메뉴나 하단으로 이동
6. 종목 카드: 추천 이유 1줄과 핵심 수치 2개만 노출
7. 신규 가입자: 관심사 설정과 알림 설정 CTA를 상단 고정
