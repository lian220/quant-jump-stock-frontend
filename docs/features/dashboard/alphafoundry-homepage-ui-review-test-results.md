## 실행한 시나리오
### TC-E2E-01: 데스크톱 비로그인 메인 화면 점검
- 결과: PASS
- 관찰:
  - 히어로 카피는 명확하지만 첫 화면이 텍스트 중심이라 서비스 신뢰 근거가 빠르게 드러나지 않는다.
  - 첫 CTA는 `오늘의 AI 추천 보기`인데, 그 아래로 AI 종목, 뉴스, 전략, 회원가입 CTA가 연속 배치되어 메인 목표가 분산된다.

### TC-E2E-02: 모바일 비로그인 메인 화면 점검
- 결과: PASS
- 관찰:
  - 390px 폭 기준에서 상단 내비게이션이 모바일 전용 패턴으로 전환되지 않고 한 줄로 유지됐다.
  - 화면이 가로로 넘치며 우측에 큰 검은 빈 영역이 생겼고, 히어로가 좌측 일부만 보였다.

### TC-E2E-03: 데스크톱 로그인 후 메인 화면 점검
- 결과: PASS

  - 건너뛰기 후 홈은 `오늘의 AI 분석` 대시보드형 화면으로 바뀌며 CTA도 `전체 분석 보기`, `전략 둘러보기`로 변경된다.
  - 같은 `/` URL이 비로그인 랜딩과 로그인 대시보드로 완전히 다른 역할을 수행한다.

### TC-E2E-04: 모바일 로그인 후 메인 화면 점검
- 결과: PASS
- 관찰:
  - 모바일 비로그인과 동일하게 헤더/레이아웃이 가로로 넘쳤다.
  - 로그인 후에도 CTA와 핵심 카드가 좌측에만 몰리고 우측에 큰 빈 영역이 남아 첫 화면 품질 인상이 크게 떨어진다.

## 발견된 이슈
### P0. 모바일 첫 화면 반응형 붕괴
- 증상:
  - 비로그인/로그인 모두 390px 폭에서 상단 헤더가 한 줄 overflow 상태로 남는다.
  - 우측에 큰 검은 빈 영역이 생기고, 핵심 콘텐츠가 좌측에 잘린 채 보인다.
- 영향:
  - 첫 방문 신뢰도가 크게 떨어지고, 모바일 사용자는 서비스가 깨졌다고 인식할 가능성이 높다.
  - CTA 가시성과 탭 정확도도 함께 저하된다.
- 관련 코드 후보:
  - `quant-jump-stock-frontend/src/components/layout/Header.tsx`
  - `quant-jump-stock-frontend/src/components/layout/BottomNav.tsx`
  - `quant-jump-stock-frontend/src/app/layout.tsx`
- 권장 조치:
  - 실제 모바일 브라우저 기준으로 헤더 브레이크포인트와 viewport 동작을 재검증한다.
  - 모바일에서는 햄버거 메뉴 또는 하단 네비 중심으로 전환하고, 전체 페이지의 가로 overflow 원인을 우선 제거한다.

### P1. 동일 URL `/`의 역할이 상태에 따라 급변
- 증상:
  - 비로그인 사용자는 마케팅 랜딩을 본다.
  - 로그인 사용자는 데이터 대시보드를 본다.
- 영향:
  - 홈의 정체성이 불안정해지고, 재방문 시 사용자의 위치 감각이 흐려진다.
  - QA 기준과 링크 공유 기준도 흔들린다.
- 관련 코드:
  - `quant-jump-stock-frontend/src/app/page.tsx`
- 권장 조치:
  - 홈을 하나의 목적에 고정하거나, 게스트 홈과 로그인 홈을 사실상 분리해 정보 구조를 재설계한다.

### P1. 로그인 직후 온보딩이 핵심 가치 확인을 가로막음
- 증상:
  - 로그인 후 바로 `오늘의 AI 분석` 홈이 아니라 온보딩 `1/3` 화면으로 이동한다.
  - `건너뛰기`가 가능해도 사용자는 로그인 완료 여부와 현재 단계를 다시 해석해야 한다.
- 영향:
  - 로그인 직후 기대하는 보상 경험이 지연된다.
  - 퍼널 이탈과 혼란 가능성이 높다.
- 권장 조치:
  - 홈 진입 후 인라인 배너/모달 형태로 점진적 온보딩을 노출하거나, 온보딩이 필요한 이유를 더 명확히 설명한다.

### P1. CTA 약속의 연속성이 약함
- 증상:
  - 비로그인 CTA는 `오늘의 AI 추천 보기`.
  - 로그인 후 CTA는 `전체 분석 보기`, `전략 둘러보기`.
- 영향:
  - 사용자가 방금 기대한 행동과 로그인 후 첫 행동이 끊긴다.
  - 서비스의 핵심 가치가 `추천 확인`인지 `전략 탐색`인지 흐려진다.
- 관련 코드:
  - `quant-jump-stock-frontend/src/app/page.tsx`
- 권장 조치:
  - `오늘의 추천 보기` → `내 추천 이어보기`처럼 상태 전환 후에도 같은 목적어를 유지하도록 CTA 퍼널을 맞춘다.

### P2. 메인 화면의 정보 구조가 혼재되어 집중도가 떨어짐
- 증상:
  - AI 주목 종목, 참고 종목, 뉴스, 전략, 회원가입 CTA, 긴 유의사항이 한 화면 흐름에 모두 들어간다.
- 영향:
  - 초보 투자자 기준으로 "지금 무엇을 해야 하는가"가 흐려진다.
  - 랜딩 페이지와 서비스 대시보드 목적이 한 화면에서 경쟁한다.
- 권장 조치:
  - 비로그인 홈은 `AI 추천 체험`, 로그인 홈은 `오늘의 분석 소비`에 맞춰 섹션 우선순위를 다시 정렬한다.

### P2. 신뢰 형성 요소가 하단으로 밀려 있음
- 증상:
  - 금융 서비스인데도 추천 근거, 업데이트 주기, 검증 방식, 리스크 원칙이 초반에 구조적으로 드러나지 않는다.
  - 법적 고지는 하단에서 길게 반복된다.
- 영향:
  - CTA 클릭 전에 "왜 믿어야 하는가"가 충분히 전달되지 않는다.
- 권장 조치:
  - 히어로 바로 아래에 신뢰 블록을 배치하고, 하단 고지는 요약 + 상세보기 구조로 정리한다.

## 전문가 관점 종합 의견
- 화면설계 관점:
  - 메인은 현재 `AI 추천 서비스의 첫 설득 화면`과 `로그인 후 작업 화면`이 한 URL에서 충돌하고 있다.
  - 특히 모바일 반응형 문제는 설계 이슈를 넘어 즉시 수정이 필요한 품질 결함이다.
- 사용성 관점:
  - 로그인 전후 흐름의 기대치가 끊기고, 온보딩이 보상 경험보다 앞서면서 행동 동선이 어색해진다.
- 신뢰/전환 관점:
  - 금융 서비스 특성상 초반 신뢰 장치가 부족하고, 섹션이 많아 CTA 집중력이 약하다.

## 최소 우선 수정 순서
1. 모바일 헤더 및 가로 overflow 수정
2. 로그인 직후 온보딩 진입 방식 재설계
3. 비로그인/로그인 CTA 문구와 목적 정렬
4. `/`의 역할 정의 재정리
5. 신뢰 블록과 법적 고지 구조 재배치
