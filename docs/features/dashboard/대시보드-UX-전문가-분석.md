## 분석 배경

### 사용자 피드백
- PC에서 로그인 대시보드의 정보 밀도가 너무 낮음
- 스크롤이 과도하게 필요
- 로그인/비로그인 뷰가 거의 동일 → 로그인 가치를 느끼기 어려움
- 다른 대시보드 서비스 대비 볼 수 있는 정보량이 부족

### 분석 참여 전문가
| 전문가 | 프레임워크 | 분석 관점 |
|--------|-----------|----------|
| Christensen | Jobs-to-be-Done | 사용자가 대시보드를 "고용"하는 이유 |
| Porter | Five Forces / Value Chain | 경쟁 포지셔닝, API 데이터 활용 |
| Godin | Purple Cow / Tribe | Remarkable 요소, 바이럴 단위 |
| Meadows | Systems Thinking | 피드백 루프, 시스템 구조 |
| Doumont | Structured Communication | 인지 부하, 정보 위계 |

## 현재 문제 진단

### 1. JTBD 불일치 (Christensen)

초보 투자자의 3가지 Job:
1. **"오늘 뭘 사야 하는지 알려줘"** — 의사결정 대행
2. **"이게 안전한 선택인지 확인해줘"** — 불안 해소
3. **"내가 지금 잘하고 있는지 보여줘"** — 진행 상황 확인

**문제**: 세 가지 Job을 수직으로 나열하여 어떤 Job도 제대로 수행하지 못함. 로그인이라는 Job을 고용할 이유가 없음.

### 2. 경쟁적 열위 (Porter)

| 요소 | 토스증권 | Danelfin | 현재 QJS | 문제 |
|------|---------|---------|---------|------|
| 첫 화면 정보밀도 | 높음 | 높음 | 낮음 | 빈 공간 과다 |
| 차별화 요소 가시성 | 즉시 | 즉시 | 스크롤 필요 | 경쟁력 미노출 |
| Above-the-fold 데이터 | 5개+ | 10개+ | 2-3개 | 정보 부족 |

**Dashboard API 저활용**: subscriptions, signals, market, aiUsage 데이터가 있지만 현재 활용도 극히 낮음.

### 3. Remarkable 요소 부재 (Godin)

- 첫 화면에 "이것 좀 봐"라고 스크린샷 공유할 만한 요소 없음
- 히어로 섹션이 마케팅 카피 → 벤치마킹 결론 "첫 화면 = 데이터"와 모순
- 필요한 Remarkable 요소:
  - "오늘의 AI 적중률: 68%" — 숫자 하나가 히어로 전체를 대체
  - 실시간 트랙레코드 차트 — "지난 30일 AI 추천 vs 코스피"
  - "지금 N명이 보고 있습니다" — 사회적 증거

### 4. 피드백 루프 부재 (Meadows)

**성공 플랫폼의 루프**:
```
사용자 방문 → 유용한 데이터 발견 → 행동(종목 클릭/구독)
    → 개인화 강화 → 더 유용한 데이터 → 재방문
```

**현재 QJS 루프**:
```
사용자 방문 → 같은 데이터 → 스크롤 → 이탈
    (개인화 없음, 행동 유도 약함, 재방문 동기 없음)
```

**핵심 레버리지 포인트**:
- signals(미읽은 알림) → **긴급성** 생성
- subscriptions(구독 전략) → **개인화** 생성
- aiUsage(백테스트 사용량) → **희소성** 생성

### 5. 인지 부하 문제 (Doumont)

- 모든 섹션이 동일한 시각적 무게 → "어디를 먼저 봐야 하는지" 판단 필요
- CTA가 위(히어로)와 아래(가입)에 분산 → 집중력 흩어짐
- AI 종목, 뉴스, 전략이 논리적 관계 없이 나열

## 전문가 합의 사항

1. **로그인/비로그인은 완전히 다른 화면이어야 함**
   - 비로그인 = **Acquisition Funnel** (전환 시스템)
   - 로그인 = **Engagement Loop** (유지 시스템)
2. **첫 화면 = 데이터, 카피 아님**
3. **Dashboard API 데이터 적극 활용 필요**
4. **PC는 multi-column, 모바일은 계층화된 single-column**
5. **PC는 "분석 세션"(길게 머무름), 모바일은 "체크인"(빠르게 확인)**

### 영역별 설계 근거

### 현재 대비 핵심 변경

- `max-w-2xl` 미니 대시보드 → 전체 너비 활용

- AI 종목 3개 → 4개 (4-column grid)
- 전략 + 뉴스 직렬 → 병렬로 스크롤 50% 감소

- Dashboard API 데이터(구독전략, 알림, 시장지수, AI사용량) 적극 활용

### 모바일 전략: "밀도 높되 계층화"

| Viewport | 포함 요소 | 완료되는 Job |
|----------|----------|-------------|
| 1st | 핵심 숫자 3개 + 미읽은 신호 | "내 상황 파악" |
| 2nd | 시장 지수 + AI 종목 (수평 스크롤) | "시장 파악" |
| 3rd | 전략 + 뉴스 | "탐색" (선택적 스크롤) |

**1st viewport만으로 "체크인" Job이 완료**되어야 함. 73.4% 이탈률은 첫 화면이 Job을 완료시키지 못해서 발생.

### 비로그인 핵심 전략

- **히어로에서 카피 제거, 데이터로 대체**: "주식, 뭘 사야 할지 모르겠다면" → "오늘 AI가 분석한 35종목 중 매수 신호 8건"
- **블러 처리로 FOMO 유도**: AI 종목 4개 중 2개 공개, 2개 블러 + 로그인 유도
- **트랙레코드 차트**: "AI가 실제로 작동한다"는 시각적 증거
- **사회적 증거**: 실시간 이용자 수, 누적 분석 건수

## 전략적 트레이드오프

### 정보 밀도 vs 초보자 이탈

- **문제**: 밀도를 높이면 73.4% 초보자 이탈 위험 증가
- **해결**: 밀도는 높이되, 각 요소를 **자기완결적**으로 설계. 카드 하나만 봐도 의미 파악 가능. 전체를 이해할 필요 없이 부분만으로 가치를 느끼는 구조.

### 개인화 깊이 vs 개발 비용

- **1단계 (즉시)**: Dashboard API 기존 데이터(subscriptions, signals, market, aiUsage) 활용 — 추가 백엔드 개발 최소
- **2단계 (중기)**: 사용 패턴 기반 추천, 포트폴리오 성과 추적

## 주의사항 (Blind Spots)

1. **빈 대시보드 (신규 가입자)**: 구독 전략 0, 신호 0, 사용량 0 → 온보딩 플로우 별도 설계 필요
2. **실시간성 범위**: 시장 지수/주가 업데이트 주기 명확화. 15분 지연이면 "실시간" 표현 지양
3. **적중률 정의**: "68% 적중률" 측정 기준 투명 공개 필요. 미정의 시 불신 유발
4. **최소 신호 빈도**: signals 데이터가 재방문 루프의 핵심인데, 신호 발생 빈도가 낮으면 "변화 없는 화면"이 됨

## 구현 우선순위 (제안)

| 순서 | 작업 | 영향도 | 난이도 | 의존성 |
|------|------|-------|-------|--------|
| 1 | PC 로그인: 2-column 레이아웃 (①②, ④⑤) | 높음 | 낮음 | 없음 |
| 2 | Dashboard API 데이터 화면 표시 (구독전략, 알림, 시장지수, AI사용량) | 높음 | 중간 | Dashboard API 연동 |
| 3 | 모바일: 핵심 숫자 3개 + 미읽은 신호 1st viewport | 높음 | 중간 | Dashboard API |
| 4 | AI 종목 카드 4-column + 정보 밀도 향상 | 중간 | 낮음 | 없음 |
| 5 | 비로그인: 데이터 히어로 (카피 → 숫자) | 중간 | 낮음 | 없음 |
| 6 | 비로그인: 블러 처리 + 로그인 유도 | 중간 | 낮음 | 없음 |
| 7 | 트랙레코드 미니 차트 | 높음 | 높음 | 성과 데이터 API 필요 |
| 8 | 빈 대시보드 온보딩 플로우 | 중간 | 중간 | UX 설계 필요 |
