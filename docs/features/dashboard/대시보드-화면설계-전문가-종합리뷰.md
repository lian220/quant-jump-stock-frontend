> *"A user interface is like a joke. If you have to explain it, it's not that good."*

### 10 Usability Heuristics 체크리스트

### 상세 위반 분석

#### H1. 시스템 상태의 가시성 — ⚠️ 중

**비로그인 PC/모바일 공통:**
- "오늘 AI가 35개 종목을 분석해서 34개를 골랐어요" → 데이터 갱신 시점이 불명확. "오늘"이 언제인지, 마지막 업데이트 시각이 없음
- 74% / 72점 통계에 기준일, 비교 기간 없음 → 사용자가 이 숫자의 맥락을 알 수 없음

**로그인 PC/모바일:**
- 동일 이슈 + 구독 전략 알림/시그널 상태가 대시보드에 전혀 노출되지 않음
- "분석 중" 등 실시간 상태 표시 없음

**개선안:**
- 헤더 또는 히어로 영역에 "마지막 분석: 2026-03-10 09:00 KST" 타임스탬프 추가
- 74% / 72점에 "(최근 30일 기준)" 라벨 명시
- 로그인 시 "읽지 않은 시그널 N건" 뱃지 추가

#### H4. 일관성과 표준 — ❌ 높음

**비로그인 → 로그인 전환 시 동일 URL(`/`)에서 완전히 다른 페이지 렌더링:**
- 비로그인: 마케팅 히어로 ("주식, 뭘 사야 할지 모르겠다면") + CTA
- 로그인: 데이터 대시보드 ("오늘의 AI 분석")
- **같은 URL이 완전히 다른 역할** → 외부 일관성 위반 (사용자가 URL 공유 시 혼란)
- 로그인/비로그인 간 시각적 연속성 없음. 하단 섹션(AI 주목 종목, 뉴스, 전략)은 동일하나, 상단이 완전히 달라 "같은 서비스인가?" 느낌

**종목 카드 내부 불일치:**
- AI 주목 종목 카드: 가격($389.32), 상승률(+24.3%), 목표가($483.95) 표시
- 참고 종목 카드: 가격($264.35) → 목표가($222) — 화살표 방향이 다름 (→ vs 아무것도 없음)
- 카드 크기, 정보량, 상호작용 패턴이 섹션마다 다름

**개선안:**
- Option A: `/` (대시보드, 로그인 필수) + `/welcome` (랜딩, 비로그인)으로 URL 분리
- Option B: 동일 URL 유지하되, 비로그인 히어로를 데이터 중심으로 전환해 시각적 연속성 확보
- 카드 컴포넌트 통일: 모든 종목 카드에 동일한 가격/목표가 표시 형식 적용

#### H8. 미적이고 미니멀한 디자인 — ❌ 높음

**PC 비로그인 (전체 페이지 스크롤 분석):**

| 스크롤 위치 | 콘텐츠 | 문제 |
|-----------|--------|------|
| Viewport 1 | 히어로 + 통계 + AI 주목 종목 상단 | ✅ 적정 (핵심 가치 전달) |
| Viewport 2 | AI 주목 종목 3카드 + 참고 종목 | ⚠️ 참고 종목이 1개뿐인데 별도 섹션 차지 |
| Viewport 3 | 면책조항 + CTA + 뉴스 | ⚠️ 면책조항이 CTA 위에 위치해 흐름 끊김 |
| Viewport 4 | 뉴스 3건 + 인기 전략 상단 | ⚠️ 뉴스가 3건뿐인데 넓은 공간 차지 |
| Viewport 5 | 인기 전략 3카드 | ⚠️ 전략 설명이 길어 스캔하기 어려움 |
| Viewport 6 | CTA 배너 + 투자 유의사항 + 푸터 | ⚠️ 유의사항이 2번 반복 (Viewport 3 + 6) |

**핵심 문제:**
1. **투자 유의사항 중복**: 본문 중간 + 푸터에 거의 동일한 면책조항이 2회 등장 → 법적으로 1회면 충분
2. **참고 종목 섹션의 비효율**: 1개 종목에 별도 헤딩 + 설명 + 여백 → 공간 낭비
3. **뉴스 섹션**: 3건 헤드라인만 있고 요약/카테고리/중요도 구분 없음 → 정보 가치 낮음
4. **페이지 총 길이**: PC 6스크롤, 모바일 8+스크롤 → 핵심 정보 도달까지 너무 깊음

**개선안:**
- 투자 유의사항을 푸터 1회로 통합, 본문에서 제거
- 참고 종목을 AI 주목 종목 카드 하단에 "접기/펼치기"로 통합
- 뉴스에 카테고리 태그 + 시장 영향도 표시 추가
- 전략 카드 설명을 2줄 이내로 truncate

#### H3. 사용자 제어와 자유 — ⚠️ 중

- 모든 종목 카드 클릭 → `/recommendations`로 이동 (개별 종목 상세가 아님)
- 사용자가 "마이크론"을 클릭하면 마이크론 상세를 기대하지만, 전체 추천 목록으로 이동
- 뒤로가기 → 홈 → 같은 카드 클릭 → 같은 목록 = **반복 루프**

**개선안:**
- 종목 카드 클릭 → `/recommendations?stock=MU` 또는 `/stocks/MU`로 개별 종목 상세 연결
- "전체 분석 보기" 버튼은 별도로 유지

#### H6. 인식 > 기억 — ⚠️ 중

- "차트 패턴 71점", "AI 예측 93점", "뉴스 반응 56점" → 이 점수의 의미/기준이 어디에도 설명 없음
- "안정성 지수 0.02" → 높은 건지 낮은 건지 맥락 없음 (Sharpe ratio?)
- 사용자가 점수 체계를 기억하고 있어야 해석 가능

**개선안:**
- 점수 옆에 미니 바/게이지 또는 색상 코딩 (빨강 0-40, 노랑 40-70, 초록 70-100)
- 첫 방문 시 점수 체계 설명 툴팁 또는 "?" 아이콘
- 안정성 지수에 "높을수록 안정" 등 방향성 라벨

#### H7. 유연성과 효율성 — ⚠️ 중

- 파워 유저를 위한 단축 경로 없음 (관심 종목 바로가기, 구독 전략 대시보드 등)
- 로그인 사용자와 비로그인 사용자가 AI 주목 종목~전략 섹션에서 동일한 경험 → 개인화 없음
- 키보드 내비게이션, 검색 바 등 효율성 도구 부재

**개선안:**
- 로그인 사용자: 구독 전략 성과 요약 + 미확인 시그널을 최상단에 배치
- 검색 바를 헤더에 추가 (종목 검색 → 해당 분석 페이지로 직행)

## 2. Steve Krug — "Don't Make Me Think" 평가

> *"Get rid of half the words on each page, then get rid of half of what's left."*

### 5초 테스트 시뮬레이션

**비로그인 PC — 첫 5초에 사용자가 인지하는 것:**

| 요소 | 인지 가능 | 이해 가능 | 평가 |
|------|----------|----------|------|
| "주식, 뭘 사야 할지 모르겠다면" | ✅ | ✅ | 명확한 가치 제안 |
| "오늘의 AI 추천 보기 →" CTA | ✅ | ✅ | 행동 유도 명확 |
| 3개 통계 배지 (35개, 804건, 참고 정보) | ✅ | ⚠️ | 글씨 작고 의미 파악에 2-3초 필요 |
| AI 주목 종목 카드 (스크롤 필요) | ❌ | - | 1st viewport 밖 |

**판정: PASS (조건부)** — 히어로 메시지는 명확하나, "이 서비스가 뭘 해주는지"의 구체적 증거(실적, 수치)가 첫 화면에 부족

**로그인 PC — 첫 5초:**

| 요소 | 인지 가능 | 이해 가능 | 평가 |
|------|----------|----------|------|
| "오늘의 AI 분석" | ✅ | ⚠️ | "분석"이 뭘 의미하는지 불명확 |
| "전체 분석 보기" / "전략 둘러보기" 버튼 2개 | ✅ | ⚠️ | 둘의 차이가 즉시 파악 안 됨 |
| 74% / 72점 통계 | ✅ | ❌ | 기준 없는 숫자 → "뭐가 74%?" |
| "35개 종목 분석해서 34개 골랐어요" | ✅ | ✅ | 가장 이해하기 쉬운 문장 |

**판정: FAIL** — "오늘의 AI 분석"이라는 제목은 모호하고, 74% / 72점에 대한 맥락이 전혀 없어 사용자가 "생각해야 함"

### "생각하게 만드는" 요소 (Think-Points)

#### Think-Point 1: "전체 분석 보기" vs "전략 둘러보기" — 뭐가 다른 거지?

**현재 상태:** 두 버튼이 나란히 있으나:
- "전체 분석 보기" → `/recommendations` (AI 분석 종목 목록)
- "전략 둘러보기" → `/strategies` (투자 전략 마켓플레이스)
- 사용자는 "분석"과 "전략"의 차이를 즉시 이해하지 못함
- 특히 초보 투자자에게 이 구분은 추상적

**Krug의 처방:**
- 버튼 라벨을 구체적으로: "AI가 고른 종목 보기" / "투자 전략 구독하기"
- 또는 두 버튼을 제거하고, 페이지 내 자연스러운 흐름으로 대체

#### Think-Point 2: "74% 좋은 평가 비율" — 뭐가 좋다는 거지?

**현재 상태:**
- "좋은 평가 비율 74%" — 뭘 평가했는지, 기간은 언제인지, 비교 대상은 뭔지 없음
- "평균 AI 점수 72점" — 100점 만점? 무슨 기준?
- 이 두 숫자가 왜 중요한지에 대한 맥락 제로

**Krug의 처방:**
- "AI가 분석한 35개 중 26개가 '매수 추천' (74%)" → 구체적 풀어쓰기
- "평균 AI 점수 72/100 (상위 30%)" → 상대적 위치 제공
- 또는 이 통계 블록을 제거하고, AI 주목 종목 카드에 직접 통합

#### Think-Point 3: 종목 카드의 3개 점수 — 어떻게 해석하지?

**현재 상태 (마이크론 카드):**
```
차트 패턴: 71점 | AI 예측: 93점 | 뉴스 반응: 56점
```
- 3개 점수의 가중치, 해석 방법, 총점과의 관계가 불명확
- "차트 패턴 71점"이 좋은 건지 보통인 건지 판단 불가
- 3개 점수의 총합 ≠ 종합 점수 74점 → 혼란

**Krug의 처방:**
- 컬러 바 추가: 초록(70+), 노랑(40-69), 빨강(0-39) → 한눈에 좋고 나쁨 판단
- 종합 점수가 3개의 평균이 아니라면, "종합 점수 산출 방법" 링크 또는 i 아이콘
- 가장 간단한 해법: 카드에 종합 점수 + "강력 추천" 라벨만 남기고, 세부 점수는 상세 페이지로 이동

#### Think-Point 4: 인기 전략 카드 — 뭘 기준으로 고르지?

**현재 상태:**
- 골든크로스: 수익률 +3.33%, 안정성 0.02
- RSI 과매도 반등: 수익률 +72.37%, 안정성 2.74
- 저PER 가치주: 수익률 N/A, 안정성 N/A

**문제:**
- +3.33% vs +72.37% → 20배 차이인데 왜 둘 다 "인기"인가?
- "안정성 지수"의 의미 불명 (0.02가 좋은 건지 2.74가 좋은 건지)
- N/A가 "아직 데이터 없음"인지 "해당 없음"인지 불명확
- 구독자 2명/1명/0명 → 실질적으로 아무도 안 쓰는 느낌

**Krug의 처방:**
- 안정성 지수를 "위험도: 낮음/중간/높음"으로 변환
- N/A를 "백테스트 진행 중" 등 명확한 상태로 교체
- 수익률 차이가 큰 전략들은 "안정형" / "공격형" 등 태그로 성격 구분
- 구독자가 적을 때는 "신규 전략" 뱃지로 전환

### Krug의 핵심 진단

> **"페이지 단어 수의 절반을 줄이고, 남은 것의 절반을 다시 줄여라"**

현재 홈페이지의 가장 큰 문제는 **설명 없는 숫자의 나열**이다. 74%, 72점, 71점/93점/56점, 0.02, 2.74... 이 숫자들이 맥락 없이 던져지면 사용자는 "생각해야 한다." 금융 서비스에서 사용자가 숫자를 해석하느라 생각하게 만드는 것은 이탈의 직접적 원인이다.

## 3. Luke Wroblewski — 모바일 퍼스트 평가

> *"Mobile forces you to prioritize. What is the most important thing?"*

### 핵심 질문: 토스증권(데이터 밀도) vs Robinhood(극단적 단순함)?

**Alpha Foundry의 정체성 분석:**

| 특성 | 토스증권 방향 | Robinhood 방향 | Alpha Foundry |
|------|-------------|---------------|---------------|
| 타겟 유저 | 중급+ 투자자 | 완전 초보자 | **초보자 + AI 의존** |
| 핵심 가치 | 데이터 접근성 | 행동 단순화 | **AI 판단 전달** |
| 정보 밀도 | 높음 | 극히 낮음 | 현재: 중간 (방향 불명) |
| 첫 화면 | 내 포트폴리오 | 관심 종목 1개 | 현재: AI 추천 3개 |

**Wroblewski의 판단:**

Alpha Foundry는 **토스증권도 Robinhood도 아닌 제3의 패턴**이 필요하다:
- **"AI가 대신 분석해주는" 서비스** → 사용자는 데이터를 직접 분석하지 않음
- 핵심은 "AI가 뭘 골랐는가" + "왜 골랐는가" + "그래서 뭘 해야 하는가"
- **Danelfin 패턴이 가장 적합**: AI 점수 중심 + 실적 증명 + 간결한 액션

→ **결론: 토스의 데이터 밀도는 과하고, Robinhood의 단순함은 부족. "AI 브리핑" 패턴으로 가야 함**

## 4. Edward Tufte — 정보 시각화 평가

> *"Above all else show the data."*

### 데이터-잉크 비율 분석

**Tufte의 원칙: 화면의 모든 픽셀은 정보를 전달하거나 제거되어야 한다**

#### AI 주목 종목 카드 분석

```
현재 카드 구성:
┌──────────────────────────────┐
│ 마이크론           74점       │ ← 정보 잉크 ✅
│ MU              강력 추천     │ ← "강력 추천"은 74점과 중복
│                              │
│ 💡 AI 예측 강세(9.3점),      │ ← 정보 잉크 ✅
│    뉴스 긍정(5.7점),         │
│    기술적 신호 2/3개 충족      │
│                              │
│ $389.32    +24.3%            │ ← 정보 잉크 ✅
│ 목표가     $483.95            │ ← 정보 잉크 ✅
│                              │
│ 차트패턴  AI예측  뉴스반응     │
│  71점     93점    56점       │ ← 숫자만, 시각적 맵핑 없음
└──────────────────────────────┘
```

**Tufte의 진단:**

1. **Chartjunk (제거 대상):**
   - "강력 추천" 뱃지: 74점이 이미 등급을 나타냄 → 중복 라벨
   - 카드 외곽 선이 과도하게 두꺼움 (정보 없는 데코레이션)
   - 💡 이모지: 장식적, 정보 전달 없음

2. **누락된 데이터 잉크:**
   - 71점/93점/56점이 **숫자만** 있고 시각적 인코딩 없음
   - 색상, 바, 게이지 등 사전 주의적(pre-attentive) 처리 가능한 시각 요소 부재
   - 현재가 → 목표가의 **상승 여력**이 퍼센트로 표시되지 않음 (계산 필요)

3. **데이터-잉크 비율 계산:**
   - 카드 면적 중 실제 데이터: ~40%
   - 나머지 60%: 여백, 배경, 라벨, 장식 → **0.4 D-I ratio** (Tufte 권장: 0.7+)

#### Sparkline(미니 차트) 설계 제안

**현재: 트랙레코드 차트 없음**

Tufte는 텍스트 크기의 작은 차트(sparkline)를 통해 맥락을 풍부하게 만들 것을 강력 권장:

```
제안: AI 종목 카드에 sparkline 추가

┌──────────────────────────────┐
│ 마이크론 MU           74점   │
│ $389.32 +24.3%  ▁▂▃▄▅▆▇▆▅▄▅ │ ← 30일 가격 sparkline
│ AI:93 | 차트:71 | 뉴스:56   │ ← 숫자 + 컬러 바
│ 목표: $483.95 (+24.3%)     │
└──────────────────────────────┘
```

**Sparkline 적용 포인트:**
- **종목 카드**: 30일 가격 추이 (위 예시)
- **전략 카드**: 백테스트 수익률 곡선
- **히어로 통계**: AI 적중률 30일 추이
- **시장 지수**: 일중 변동 미니 차트

#### 정보 밀도 비교: 현재 vs 제안

| 메트릭 | 현재 | 제안 | 개선율 |
|--------|------|------|--------|
| 1st viewport 데이터 포인트 (PC, 로그인) | 6개 | 15+개 | +150% |
| 종목 카드 D-I ratio | 0.4 | 0.7 | +75% |
| 스크롤 없이 비교 가능한 종목 수 | 0개 (카드 열기 필요) | 4개 | +∞ |
| 시각적 인코딩 수 (색상/크기/위치) | 1 (색상) | 4 (색상+크기+위치+형태) | +300% |

### Small Multiples 적용

**현재 AI 주목 종목 3카드: 각 카드가 독립적으로 정보를 전달 → 비교 불가**

Tufte의 Small Multiples 원칙 적용:
```
┌────────────────┬────────────────┬────────────────┐
│ 마이크론  74점  │ 세일즈포스 74점 │ 엔비디아  73점  │
│ ▁▂▃▅▇▅▃▂▃▅▇▆  │ ▅▆▇▇▆▅▅▆▇▇▆▅  │ ▃▅▆▇▆▅▃▂▃▅▆▇  │
│ $389 → $484   │ $199 → $255   │ $183 → $204   │
│ ████████░░ 93  │ ██████████ 100 │ █████████░ 88  │ ← AI 예측 바
│ ███████░░░ 71  │ █████░░░░░ 57  │ ███████░░░ 71  │ ← 차트 패턴 바
│ █████░░░░░ 56  │ █████░░░░░ 56  │ █████░░░░░ 58  │ ← 뉴스 반응 바
└────────────────┴────────────────┴────────────────┘
  같은 축, 같은 스케일 → 한눈에 비교 가능
```

이렇게 하면:
- 세일즈포스의 AI 예측이 100점으로 가장 높다는 것이 **바의 길이로 즉시** 인지됨
- 세일즈포스의 차트 패턴이 57점으로 약하다는 것이 **다른 카드와 비교로** 드러남
- 현재는 숫자만 읽고 머리로 비교해야 함

### 불필요한 Chartjunk 제거 대상

| 요소 | 위치 | 문제 | 제안 |
|------|------|------|------|
| "강력 추천" 뱃지 | 종목 카드 | 74점과 중복 | 점수 색상으로 등급 표현 |
| "차트 패턴 + AI 예측 종합 분석" 라벨 | AI 주목 종목 섹션 | 매번 보이는 설명 | 첫 방문만 표시 또는 제거 |
| 💡 이모지 | 카드 설명 | 장식적 | 제거 |
| 🔥 이모지 | 섹션 제목 | 장식적 | 제거 또는 아이콘으로 교체 |
| "위 정보는 AI가..." 면책조항 | AI 종목 하단 | 페이지 중간에 흐름 끊김 | 푸터로 이동 |
| "투자 유의사항" 2회 반복 | 본문 + 푸터 | 중복 | 푸터 1회로 통합 |

## 5. Nir Eyal — Hooked 모델 / 온보딩 평가

> *"The Hook Model describes an experience designed to connect the user's problem to your solution with enough frequency to form a habit."*

### Hook 모델 4단계 평가

```
Trigger → Action → Variable Reward → Investment
  ↑                                      │
  └──────────────────────────────────────┘
```

#### 1단계: Trigger (트리거)

**External Trigger 현황:**
- ⚠️ 이메일 알림: 미확인 (설정 화면에 존재 여부 불명)
- ⚠️ 푸시 알림: PWA 설치 유도는 있으나, 실제 푸시 구현 미확인
- ❌ SMS/카카오톡 알림: 없음
- ❌ 일일 브리핑: "오늘의 AI 분석 완료" 알림 없음

**Internal Trigger:**
- 목표: "주식 뭘 사야 하지?" → alphafoundry.app 접속 습관
- ❌ 현재 이 습관 루프를 강화하는 장치 없음

**진단:** 트리거가 매우 약함. 사용자가 **스스로 기억해서** 방문해야 함.

**개선안:**
- [ ] 일일 AI 분석 완료 시 푸시 알림 (09:00 KST)
- [ ] 구독 전략 시그널 발생 시 즉시 알림
- [ ] 주간 AI 성과 요약 이메일
- [ ] "오늘 AI가 새로운 종목을 발견했어요" — 호기심 트리거

#### 2단계: Action (행동)

**Fogg의 B=MAP (Behavior = Motivation × Ability × Prompt)**

| 요소 | 현재 수준 | 문제 |
|------|----------|------|
| **Motivation** (동기) | 중 | "AI가 종목을 골라준다"는 매력적이나, 신뢰 증거 부족 |
| **Ability** (능력) | 중-하 | 숫자 해석 필요, 전문 용어 존재 (RSI, PER, 골든크로스) |
| **Prompt** (촉발) | 하 | 외부 트리거 부재, 재방문 유인 약함 |

**핵심 행동(Core Action)이 불명확:**
- 홈에서 사용자가 해야 할 **하나의 행동**이 뭔가?
- 비로그인: "회원가입" → 명확 ✅
- 로그인: "???" → **불명확** ❌
  - 종목 카드 클릭? 전체 분석 보기? 전략 구독?
  - 3개 이상의 선택지가 동등한 시각적 무게로 제시됨

**개선안:**
- 로그인 대시보드의 Core Action 정의: **"오늘의 AI 브리핑 확인"**
- 대시보드 최상단에 "오늘의 브리핑" 카드 1개 (신규 시그널 + 핵심 변화 요약)
- 다른 섹션은 브리핑 이후의 탐색 경로로 위계 낮춤

#### 3단계: Variable Reward (가변 보상)

**현재 보상 구조:**
- 종목 추천 내용 (매일 다른 종목) → ✅ 가변적
- AI 점수 (매일 변동) → ✅ 가변적
- 뉴스 헤드라인 (매일 다름) → ✅ 가변적

**문제: 보상의 "질"이 낮음**
- 종목 추천을 보고 나서 → "그래서 뭘 하라고?" → 행동 연결 없음
- 전략을 구독해도 → 대시보드에서 성과가 안 보임 → 보상 피드백 끊김
- "내가 이 서비스를 써서 얼마나 이득을 봤는지" 추적 불가

**개선안:**
- [ ] "AI 추천 따라갔으면" 시뮬레이션: "30일 전 AI 추천 포트폴리오 → +X% 수익"
- [ ] 구독 전략 실시간 성과 피드백: "골든크로스 전략: 이번 달 +2.1%"
- [ ] 개인 트랙레코드: "당신이 관심 등록한 종목들의 이후 성과"

#### 4단계: Investment (투자/관여)

**사용자가 서비스에 "저장"하는 것:**
- ⚠️ 전략 구독 (있으나 대시보드에 미반영)
- ❌ 관심 종목 (기능 유무 불명)
- ❌ 분석 히스토리 (과거 AI 추천 추적 불가)
- ❌ 개인 설정 (분석 스타일, 관심 섹터 등)

**진단:** Investment가 거의 없음 → 서비스 전환 비용이 0에 가까움

**개선안:**
- [ ] 관심 종목 리스트 (워치리스트) + 대시보드에 표시
- [ ] AI 분석 히스토리 (과거 추천 → 실제 결과 추적)
- [ ] 투자 성향 프로필 (보수적/공격적 → AI 추천 필터)

### Cold Start (빈 대시보드) 문제

**신규 가입자의 첫 로그인 대시보드:**

```
구독 전략: 0개
시그널 알림: 0건
관심 종목: 0개
사용량: 0/10
```

**현재 상태:** 비로그인과 거의 동일한 화면 → "가입한 의미가 뭐지?"

**Nir Eyal의 처방 — "강제 온보딩" vs "기본값 채움":**

| 전략 | 장점 | 단점 | Alpha Foundry 적합도 |
|------|------|------|---------------------|
| **강제 온보딩** (3단계 설정 마법사) | 개인화 높음, 데이터 수집 | 이탈율 증가, 귀찮음 | ⚠️ 중간 |
| **기본값 채움** (자동 구독/추천) | 즉시 가치 체험, 낮은 마찰 | 개인화 낮음, 불필요한 알림 | ✅ 높음 |
| **하이브리드** (기본값 + 점진적 온보딩) | 즉시 가치 + 점진적 개인화 | 구현 복잡 | ✅ 최적 |

**추천: 하이브리드 전략**
1. 가입 즉시: 인기 전략 1개 자동 구독 + AI 추천 Top3 관심 종목 자동 등록
2. 첫 방문 대시보드: "AI가 당신을 위해 준비했어요" + 기본 설정된 내용 표시
3. 3일 후: "투자 성향을 알려주면 더 정확한 추천이 가능해요" 점진적 프로필 완성
4. 7일 후: "이번 주 AI 추천 성과: +X%" → 첫 Variable Reward 제공

### "아하 모먼트"까지의 최단 경로

**현재 경로 (7단계):**
```
가입 → 로그인 → 홈 → AI 주목 종목 확인 → 전략 페이지 이동
→ 전략 이해 → 구독 → (다음 날) 시그널 확인
```
- 시간: 최소 24시간+ (시그널이 발생해야 가치 체험)
- 이탈 지점: 3단계 (홈에서 뭘 해야 할지 모름)

**제안 경로 (3단계):**
```
가입 → 개인화된 대시보드 (기본값 채움) → "AI가 오늘 추천한 종목과 이유" 확인
```
- 시간: 30초
- 아하 모먼트: "AI가 나를 위해 이미 일하고 있구나"

## 6. Julie Zhuo — 데이터 기반 디자인 평가

> *"Data tells you what; research tells you why."*

### A/B 테스트로 검증해야 할 핵심 가설 3가지

#### 가설 1: 데이터 중심 히어로 vs 감성 히어로

```
Control (현재):     "주식, 뭘 사야 할지 모르겠다면"
Variant A:          "오늘 AI가 35개 분석 → 8개 매수 추천 (적중률 68%)"
Variant B:          "지난달 AI TOP3: +12.4% (NASDAQ +3.2%)"
```

**측정 지표:**
- Primary: 회원가입 전환율 (CTA 클릭 → 가입 완료)
- Secondary: 히어로 아래 스크롤 비율, 종목 카드 클릭률
- Guard: 이탈률 (변경으로 인한 혼란 없는지)

**예상:** Variant A가 더 높은 전환율. 구체적 숫자가 신뢰감을 줌.
**위험:** 적중률이 낮은 날 역효과 → 동적 표시 시 하한선 설정 필요

#### 가설 2: 종목 카드 정보량 — 풀 카드 vs 미니 카드

```
Control (현재):     종목명 + 점수 + 가격 + 3개 세부점수 + 설명
Variant A:          종목명 + 종합점수 + 가격 + sparkline (세부는 클릭 시)
Variant B:          종목명 + "강력 추천" 라벨 + 1줄 요약만
```

**측정 지표:**
- Primary: 종목 카드 → 상세 페이지 클릭률 (CTR)
- Secondary: 카드 영역 체류 시간, 스크롤 깊이
- Guard: 전체 페이지 이탈률

**예상:** Variant A가 CTR 최적. 적절한 정보 + 호기심 갭.
**위험:** 정보가 너무 적으면 신뢰도 하락 (금융 서비스 특성)

#### 가설 3: 로그인 대시보드 — 개인화 섹션 유무

```
Control (현재):     범용 AI 분석 대시보드 (비로그인과 동일한 콘텐츠)
Variant A:          최상단에 "내 전략 성과 + 미확인 시그널" 개인화 블록 추가
Variant B:          "오늘의 AI 브리핑" 카드 1개 (요약 + 핵심 변화)
```

**측정 지표:**
- Primary: 로그인 유저 재방문율 (D1, D7, D30 retention)
- Secondary: 전략 구독 전환율, 시그널 확인율
- Guard: 페이지 로딩 시간 (API 추가로 인한 성능 영향)

**예상:** Variant A가 retention 향상. "내 것"이 보이면 재방문 동기 생김.
**위험:** Dashboard API 응답 시간이 느리면 역효과

### 정성적 vs 정량적 검증 분류

| 검증 항목 | 방법론 | 이유 |
|----------|--------|------|
| 히어로 메시지 효과 | **정량** (A/B 테스트) | 전환율로 직접 측정 가능 |
| 종목 카드 정보량 | **정량** (A/B 테스트) | CTR로 직접 측정 가능 |
| 개인화 섹션 효과 | **정량** (A/B 테스트) | Retention으로 측정 |
| 점수 체계 이해도 | **정성** (사용자 인터뷰 5명) | "74점이 뭔지 이해하시나요?" |
| 전략 카드 의사결정 | **정성** (사고발화 프로토콜) | 어떤 정보로 구독 결정하는지 |
| 첫 방문 경험 | **정성** (5초 테스트 + 인터뷰) | 첫 인상, 가치 인지 확인 |
| 모바일 터치 패턴 | **정량** (히트맵/세션 녹화) | 실제 터치 위치, 스크롤 깊이 |
| 이탈 지점 | **정량** (퍼널 분석) | 어디서 떠나는지 |
| "안정성 지수" 해석 | **정성** (카드 소팅) | 용어 이해도 테스트 |
| 페이지 간 네비게이션 | **정량+정성** (태스크 완료율 + 인터뷰) | 원하는 정보 찾기 효율 |

### Julie Zhuo의 핵심 조언

> **"가장 위험한 가정부터 테스트하라."**

이 대시보드에서 가장 위험한 가정 3가지:
1. **"사용자가 AI 점수를 이해하고 신뢰한다"** → 만약 아니라면, 서비스의 핵심 가치 제안 붕괴
2. **"로그인 사용자가 매일 돌아온다"** → 만약 아니라면, 구독 모델 작동 안 함
3. **"종목 카드 3개가 첫 화면에서 충분한 가치를 전달한다"** → 만약 아니라면, 스크롤 없이 이탈

→ **1번이 가장 위험**: 사용자 5명 대상 "이 점수가 뭘 의미하는 것 같나요?" 인터뷰 선행 필수
